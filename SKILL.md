# Quant Analyzer (A股量化交易系统复盘助手)

## 技能描述
此技能用于每天收盘后，通过 **多智能体协同（Multi-Agent Orchestration）** 模式，自动对量化交易系统的运行日志进行全方位、深度诊断。系统会将任务分发给不同的专属子代理（Sub-agents），结合宏观市场、个股深度基本面/技术面，以及统计大局观，最终生成一份具备机构级专业度的策略优化报告。

## 触发条件
- 用户的 Cron 定时任务触发（如每天 17:00）。
- 用户手动发送交易日志并要求：“执行 quant-analyzer 分析一下今天的交易”。

## 工作流 (SOP) —— 多智能体协同作战方案

作为主控节点（基金经理），我将在收到日志后，启动以下子代理并行工作：

### 文件系统架构与读取权限声明
为了保证执行效率与上下文纯净，系统遵循严格的读写权限划分：
1. **静态基准 (`strategy.md`)**：存储策略核心逻辑。**Agent B, C, D** 均需读取以获取判断标准。
2. **每日动态日志 (`logs/` 文件夹)**：存放每日生成的原始日志（如 `trading_YYYY-MM-DD.log`）。**Agent B 和 D** 仅被授权读取当日最新日志；**Agent C** 拥有整个 `logs/` 目录的完整读取与代码遍历权限。
3. **滚动记忆库 (`memory.md`)**：每日复盘后 Append（追加）写入的长期记忆文件，记录核心教训。**所有 Agent** 均有读取权限，以确保分析的连贯性。

### 🧑‍💻 Agent A：宏观与主线研究员 (Macro & Sector Analyst)
- **任务**：使用 `web_search` 搜索当日“A股大盘走势”、“涨跌停家数”、“领涨/领跌板块”及“市场核心驱动事件”。
- **目标**：为整个复盘定下市场基调。明确当天的资金主攻方向，以此作为评价策略表现（是否偏离主线）的宏观参照物。

### 🕵️‍♂️ Agent B：合规与纪律审查员 (Compliance & Execution Checker)
- **任务**：读取 `strategy.md`（系统既定规则）和当日日志。
- **目标**：逐笔核对当日买卖操作。重点审查：SAR 止损线是否严格单调递增？买入标的是否符合 `extra_only` 设定的约束？是否发生了异常的滑点、抢跑或信号漂移？发现任何底层执行 Bug 立即上报。

### 📊 Agent C：量化统计与数据探索数据科学家 (Data Scientist & Quantitative Explorer)
- **独特上下文**：拥有独占的量化沙盒环境（带 Python 代码执行权限），直接对接目录下的所有**历史原始日志（Raw Logs）**。
- **任务**：**不设固定模板，自由发挥！** 每天根据输入的最新日志，自行编写 Python 探索脚本遍历历史数据。Agent C 需要像数据科学家一样，主动寻找数据中的异常、规律和相关性。例如：探索持仓周期与盈亏的散点分布、胜率随大盘指数波动的关系、触发 SAR 陷阱的特征群聚、甚至是资金分配算法的实际效率。通过清洗、计算和对比（如动态胜率、盈亏比、回撤、滑点等），抛出具有统计学意义的、出人意料的量化洞察结论。

### 🔬 Agent D：个股深度投研员 (Stock Deep-Dive Specialist)
- **任务**：聚焦系统今天**买入、卖出和持仓的核心股票**。
- **操作**：针对每一只重点标的，调用 `web_search` 抓取最新资讯、公司公告、行业研报；同时结合行情数据，计算其短期的资金面与技术面（如支撑位、放量情况等）。
- **目标**：不再是走马观花，而是像人类研究员一样，彻底搞清楚“这只股票今天到底发生了什么”，解释它为什么能赚钱，或者为什么破位止损。

### 🏆 主控汇编：最终复盘报告生成
在收集完 Agent A, B, C, D 的独立报告后，我将进行全局交叉验证，最终输出结构化报告：
1. **【市场全景与策略位置】**（来自 Agent A & C）
2. **【系统执行与纪律诊断】**（来自 Agent B）
3. **【核心个股深度剖析】**（来自 Agent D 的精细化投研）
4. **【量化优化行动指南】**（统筹 C 的统计建议与 D 的标的分析，提出权重、因子调整意见）
5. **【自动追加历史记忆】**：将主控提取的核心教训、统计发现（胜率/回撤异动）和明日观察重点，按日期结构 Append 追加写入 `memory.md` 中。
